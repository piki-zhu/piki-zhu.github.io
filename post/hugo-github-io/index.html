Hugo+Github Io 博客搭建
  • May 15, 2019

前言

第一次尝试写博客，第一篇就把搭建过程记录下来，不失为是一个好的开端。


环境安装

Git

Go

Hugo

为了方便使用，可以把Go和Hugo的安装路径配置到环境变量中。


生成站点

可以直接使用hugo命令快速生成站点：

hugo new site ./site

在当前目录新建site文件夹，生成初始站点。


新建文章

进入生成的site目录，创建一篇about文章：

hugo new about.md

about.md会生成在content/about.md，打开看下内容：

---
title: "Post"
date: 2019-07-16T16:34:12+08:00
draft: true
---

默认内容是Markdown格式，可以根据喜好切换，本博客使用的也是Markdown格式书写。


安装主题

可以在Hugo主题列表中挑选一个喜欢的皮肤（挑主题总是最快乐的时光）。以本博客使用的AllinOne主题为例，进入site目录中的themes目录，把主题git clone下来：

git clone https://github.com/orianna-zzo/AllinOne.git

主题作者在Github中详细描述了使用方法，在跑通Demo后，请务必认真阅读。


运行Hugo

在site站点根目录执行Hugo命令：

hugo server --theme=AllinOne --buildDrafts

在浏览器中打开：http://localhost:1313

恭喜你，你的第一个博客已经运行成功。

由于AllinOne主题作者的定制开发，需要将site/themes/AllinOne/exampleSite中的config.toml文件覆盖site根目录中的同名文件才能运行成功。AllinOne的GitHub中有详细使用说明。

如果你的运行命令报错了，主题的使用说明或许能帮到你。


部署到GitHub

这里介绍如何将Hugo博客部署到GitHub Pages，也是部署博客最方便的形式。

首先在GitHub上新建一个Repository用于部署博客，命名格式为：[github用户名].github.io，本博客是piki-zhu.github.io。

在site根目录执行Hugo打包命令：

hugo --theme=AllinOne --baseUrl="http://piki-zhu.github.io/"

Hugo会将打包的静态页面生成到public目录，将public目录中的所有文件push到刚刚创建的Repository的master分支。

过段时间后，访问http://piki-zhu.github.io/。

你的博客已经部署成功了~快和网友们分享你的知识和收获吧~
